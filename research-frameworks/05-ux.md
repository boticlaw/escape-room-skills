# 05 - UX en Escape Rooms
## Cómo viven la experiencia los jugadores: usabilidad y fricción cero

---

## 📋 Resumen Ejecutivo

UX (User Experience) en escape rooms es el **diseño invisible**: cuando funciona, los jugadores no lo notan; cuando falla, rompe la inmersión. Se trata de eliminar **fricción innecesaria** (no el desafío, sino los obstáculos artificiales), hacer interacciones intuitivas y asegurar que la experiencia sea **accesible** para todos.

---

## 🎯 Principios Clave

### 1. **Fricción Cero (Excepto en Puzzles)**
- **Fricción mala:** Candados atascados, props que no funcionan, reglas confusas
- **Fricción buena:** Puzzles desafiantes, deducción necesaria
- **Regla:** Eliminar toda fricción que no sea parte del puzzle

### 2. **Feedback Inmediato y Contextual**
- Cada acción debe tener respuesta clara (click, luz, sonido)
- Jugadores nunca deben preguntar "¿Funcionó?"
- Validación objetiva: El candado se abre o no, no hay "casi"

### 3. **Intuitividad (Affordances)**
- Objetos deben "decir" cómo usarse (forma, color, contexto)
- Objetos similares se usan juntos (pistas visuales consistentes)
- No requiren instrucciones escritas para interacciones básicas

### 4. **Onboarding Efectivo**
- Briefing máximo 2-3 minutos
- Reglas claras y mínimas (no lista de 20 prohibiciones)
- Objetivo principal en una frase
- "Llamada a la acción" inmediata (entran a jugar ya)

### 5. **Accesibilidad por Defecto**
- Diseñar para diversidad de habilidades desde el inicio
- Alternativas para puzzles sensoriales (audio, visual, táctil)
- Espacio físico accesible (sillas de ruedas, alturas variadas)

### 6. **Consistencia de Interacción**
- Si un candado funciona de una manera, todos igual
- Colores significativos consistentes (rojo = importante, etc.)
- Lógica interna del mundo se mantiene

## 🔧 Elementos de UX Aplicados

### **1. Signaling y Wayfinding**
| Elemento | Función UX | Ejemplo |
|----------|-----------|---------|
| **Iluminación focal** | Dirigir atención | Spotlight en objeto importante |
| **Color coding** | Agrupar elementos | Objetos azules van juntos |
| **Orden lógico** | Números, flechas | Puzzle 1 → Puzzle 2 → Puzzle 3 |
| **Feedback espacial** | Áreas desbloqueadas | Puerta abierta = progreso visible |

### **2. Reducción de Carga Cognitiva**
- **Limitar opciones activas:** Máximo 3-4 puzzles disponibles simultáneamente
- **Agrupar información:** Pistas para un puzzle cerca del puzzle
- **Progreso visible:** Mostrar cuántos faltan (3/5 candados abiertos)
- **Evitar red herrings excesivos:** Máximo 1-2 objetos decorativos por área

### **3. Ergonomía Física**
| Aspecto | Consideración | Implementación |
|---------|---------------|----------------|
| **Altura** | Objetos accesibles para todos | Puzzles a 1-1.5m, no en suelo ni techo |
| **Espacio** | Movimiento fluido | Mínimo 1m² por jugador |
| **Luz** | Leer sin esfuerzo | 300 lux mínimo en zonas de puzzle |
| **Sonido** | Claridad sin gritar | Acústica que permita conversación normal |

### **4. Diseño de Hints Naturales**
- **Hints integrados en mundo:** Libro con pista, no post-it del GM
- **Progresión de hints:** Nivel 1 (dirección), Nivel 2 (acción), Nivel 3 (casi solución)
- **Sistema de petición:** Botón físico, walkie-talkie, pantalla
- **Timing:** Disponible después de 5 min sin progreso

### **5. Accesibilidad Inclusiva**
| Discapacidad | Adaptación | Ejemplo |
|--------------|-----------|---------|
| **Visual** | Audio descriptions, braille, texturas | Puzzle con relieve, audio pista |
| **Auditiva** | Subtítulos, luces, vibraciones | Video con CC, luz parpadea = pista |
| **Motora** | Alturas variadas, alternativas físicas | Puzzle digital si no puede girar |
| **Cognitiva** | Opciones de dificultad, hints extra | Modo "relaxed" con más tiempo |
| **Neurodivergencia** | Espacio quiet, evitar overloads | Sin luces estroboscópicas, zona de calma |

## ⚠️ Errores Comunes de UX

### 1. **Briefing Excesivamente Largo**
- **Problema:** 10 minutos de reglas = olvidan todo
- **Solución:** 2-3 minutos, reglas mínimas, repetir solo lo crítico

### 2. **Props que No Funcionan**
- **Problema:** Candado atascado, sensor no detecta, mecanismo roto
- **Solución:** Testing diario, mantenimiento preventivo, backup manual

### 3. **Falta de Feedback**
- **Problema:** Jugadores no saben si solución fue correcta
- **Solución:** Feedback inmediato (sonido, luz, apertura física)

### 4. **Puzzles Ambiguos**
- **Problema:** Múltiples interpretaciones posibles
- **Solución:** Una solución clara, auto-validable

### 5. **Red Herrings Excesivos**
- **Problema:** Demasiados objetos decorativos = confusión
- **Solución:** Decoración claramente decorativa, o cada objeto tiene propósito

### 6. **Iluminación Insuficiente**
- **Problema:** No pueden leer pistas, fatiga visual
- **Solución:** 300+ lux en zonas de lectura, linterna disponible

### 7. **Espacio Reducido**
- **Problema:** 6 personas en 10m² = caos, frustración
- **Solución:** Mínimo 1.5m² por jugador, áreas diferenciadas

### 8. **No Diseñar para Accesibilidad**
- **Problema:** Excluye jugadores con discapacidades
- **Solución:** Alternativas para cada tipo de puzzle, comunicación previa

## ✅ Checklist de Validación UX

### **Onboarding**
- [ ] ¿El briefing dura menos de 3 minutos?
- [ ] ¿El objetivo principal está en una frase?
- [ ] ¿Las reglas son mínimas y claras?
- [ ] ¿Hay llamada a la acción inmediata ("¡Entren!")?

### **Interacciones**
- [ ] ¿Los objetos indican cómo usarse (affordances)?
- [ ] ¿Hay feedback inmediato en cada acción?
- [ ] ¿Las validaciones son objetivas (no ambiguas)?
- [ ] ¿La consistencia se mantiene (lógica interna)?

### **Espacio Físico**
- [ ] ¿Hay mínimo 1.5m² por jugador?
- [ ] ¿Los puzzles están a alturas accesibles (1-1.5m)?
- [ ] ¿La iluminación permite leer sin esfuerzo (300+ lux)?
- [ ] ¿El espacio es navegable para sillas de ruedas?

### **Accesibilidad**
- [ ] ¿Hay alternativas para puzzles visuales (audio/táctil)?
- [ ] ¿Hay alternativas para puzzles auditivos (visual)?
- [ ] ¿Los hints están disponibles en múltiples formatos?
- [ ] ¿Se ha comunicado accesibilidad previamente a jugadores?

### **Fricción**
- [ ] ¿Todos los props funcionan correctamente (testing diario)?
- [ ] ¿Los candados/mechanismos no se atascan?
- [ ] ¿No hay "casi soluciones" (todo es correcto o incorrecto)?
- [ ] ¿El sistema de hints es fácil de usar?

## 💡 Ejemplos de Implementación

### **Ejemplo 1: Briefing UX-Optimizado**
```
ESTRUCTURA (2:30 minutos):

1. CONTEXTO (45 seg)
"Es 1923. Elena García desapareció hace 100 años.
Su oficina sellada será demolida en 60 minutos."

2. OBJETIVO (20 seg)
"Descubran qué le pasó antes de que sea tarde."

3. REGLAS (45 seg)
- "No usen fuerza física"
- "Todo lo que necesitan está en la sala"
- "Pedid hints si os bloqueáis > 5 minutos"

4. LLAMADA A LA ACCIÓN (10 seg)
"¡Entren! El tiempo empieza ahora."

TOTAL: 2:00 minutos (margen de 30 seg para preguntas)
```

### **Ejemplo 2: Sistema de Feedback Multimodal**
```
PUZZLE: Caja fuerte de 4 dígitos

FEEDBACK VISUAL:
- LED verde se enciende al acertar
- Pantalla muestra "ACCESO CORRECTO"

FEEDBACK AUDITIVO:
- Click satisfactorio al introducir código correcto
- Beep de error si código incorrecto (no frustrante, informativo)

FEEDBACK FÍSICO:
- Caja se desbloquea con mecanismo audible
- Tapa se abre automáticamente

RESULTADO:
Jugadores nunca dudan si acertaron o no
```

### **Ejemplo 3: Accesibilidad por Defecto**
```
PUZZLE: Secuencia de sonidos (originalmente solo auditivo)

VERSIÓN INCLUSIVA:
1. Audio claro con secuencia de notas musicales
2. Pantalla con visualización de ondas de sonido
3. Vibrador que pulsa en ritmo de las notas
4. Hoja con notación musical simplificada

TODOS pueden resolverlo:
- Personas sordas usan visualización o vibrador
- Personas ciegas usan audio o vibrador
- Personas con discapacidad cognitiva usan hoja simplificada

DISEÑO: No es "adaptación", es diseño desde el inicio
```

### **Ejemplo 4: Reducción de Carga Cognitiva**
```
ESCENARIO: Sala con 10 objetos visibles

MAL DISEÑO (Carga alta):
- 8 objetos son decorativos
- 2 son pistas reales
- Jugadores pierden 15 min examinando decoración

BUEN DISEÑO (Carga optimizada):
- 3 objetos son pistas claras (iluminación focal)
- 2 objetos son decoración obvia (estilo diferente, sin brillo)
- 5 objetos tienen propósito narrativo pero no son puzzles

RESULTADO:
Jugadores enfocan energía en lo importante, no en filtrar ruido
```

## 🛠️ Aplicación al Template

### **Cambios Concretos para `templates/escape-room/`**

#### 1. **UX.json - Perfil de Usabilidad**
```json
{
  "onboarding": {
    "briefing_duracion_segundos": 150,
    "objetivo_principal": "Una frase clara",
    "reglas_maximas": 3,
    "llamada_accion": "Inmediata"
  },
  
  "accesibilidad": {
    "visual": {
      "alternativas_audio": true,
      "braille_disponible": true,
      "texturas": true
    },
    "auditiva": {
      "subtitulos": true,
      "luz_feedback": true,
      "vibracion": false
    },
    "fisica": {
      "espacio_silla_ruedas": true,
      "alturas_accesibles": true,
      "alternativas_digitales": true
    }
  },
  
  "espacio_fisico": {
    "metros_cuadrados_min": 15,
    "metros_por_jugador": 1.5,
    "iluminacion_min_lux": 300,
    "navegacion_clara": true
  },
  
  "feedback": {
    "tipos": ["visual", "auditivo", "fisico"],
    "inmediatez_segundos": 2,
    "validacion_objetiva": true
  },
  
  "friccion": {
    "testing_diario_props": true,
    "mantenimiento_semanal": true,
    "backup_manual_disponible": true
  }
}
```

#### 2. **Validador de UX**
```python
def validar_ux(juego):
    ux = juego["ux"]
    
    # Verificar espacio físico
    jugadores_max = juego["jugadores"]["max"]
    espacio_min = ux["espacio_fisico"]["metros_cuadrados_min"]
    espacio_por_jugador = espacio_min / jugadores_max
    
    if espacio_por_jugador < 1.5:
        print(f"⚠️  Espacio insuficiente: {espacio_por_jugador:.1f}m²/jugador (mín 1.5m²)")
    
    # Verificar briefing
    if ux["onboarding"]["briefing_duracion_segundos"] > 180:
        print(f"⚠️  Briefing muy largo: {ux['onboarding']['briefing_duracion_segundos']}s (máx 180s)")
    
    # Verificar accesibilidad
    for tipo, opciones in ux["accesibilidad"].items():
        if not any(opciones.values()):
            print(f"⚠️  Sin alternativas para {tipo}")
    
    return True
```
